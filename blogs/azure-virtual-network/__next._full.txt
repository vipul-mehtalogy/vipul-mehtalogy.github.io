f:T9db,<h1>Virtual Network</h1>
<p>Create a logically isolated network on Azure cloud. Allows to securely connect it to on-premises data centers or a single machine using an IPsec connection. </p>
<ul>
<li>Does not cost anything</li>
<li>Extend data center in the cloud</li>
<li>Build distributed applications</li>
<li>Remotely debug applications</li>
</ul>
<p>Create Virtual Netowrk</p>
<ul>
<li>Need to add to a subscription, resource group assign a name, and region</li>
<li>virtual network has a range of IP address, we need to specify a fixed range of IP address using <a href="/2020/11/what-is-cidr.html">CIDR</a> notations. </li>
<li>Virtual networks required a minimum of one subnet. a subnet is a division inside the IP address range.</li>
</ul>
<p> </p>
<h2>Virtual network to virtual network connection</h2>
<ul>
<li><p>Devices connected to one virtual network will not be able to connect with devices in another virtual network. There is no vnet to vnet connection by default.</p>
</li>
<li><p>Communication can be possible via a network gateway. network gateway is a site-to-site VPN. It is a virtual device. It encrypts the traffic between vnets through a tunnel. </p>
</li>
<li><p>Create a virtual network gateway</p>
</li>
<li><p>Choose a subscription, region, gateway type (VPN| ExpressRoute), VPN type (route based | policy-based), sku, vnet, choose public IP address.</p>
</li>
<li><p>Creation on virtual network gateways takes up to 45-60 min.</p>
</li>
<li><p>Each virtual network will need a separate Virtual Network Gateway.</p>
</li>
<li><p>Establish site to site connection between vnets</p>
</li>
<li><p>Go to virtual network gateway --&gt; Settings --&gt; Connections --&gt; Add</p>
</li>
<li><p>Assign name, vnet 1 &amp; vnet 2, IKE protocol (IKEv1 | IKEv2)</p>
</li>
<li><p>shared key: used to establish encryption for the connection. Should be the same on both virtual networks.</p>
</li>
<li><p>Pricing: based on the amount of time the gateway is provisioned and avaialable. data transfer inbound is free, outbound data transfer is chargable.</p>
</li>
</ul>
<h2>Virtual Network peering</h2>
<ul>
<li>Go to virtual network --&gt; Settings --&gt; Peerings --&gt; Add</li>
<li>Assign name, subscription, another vnet for peering, traffic direction, and forwarding </li>
<li>Peering is suitable for the cloud to cloud vnet connection.</li>
<li>Charged only for outbound data transfer.</li>
<li>Enable traffic farwording</li>
<li>Save the cost of network gateway</li>
<li>Quick and easy</li>
</ul>
