<p>Angular application architecture  </p>
<h2>Component</h2>
<ul>
<li><p>Control a part of UI</p>
</li>
<li><p>Defined using the @Component decorator</p>
</li>
<li><p>Contains</p>
</li>
<li><p>template: loads view for the component</p>
</li>
<li><p>Stylesheet: define look and feel</p>
</li>
<li><p>Class: business logic for component</p>
</li>
<li><p>ng generate component productList</p>
</li>
</ul>
<h2>Module</h2>
<ul>
<li><p>Group component </p>
</li>
<li><p>Module decides whether the components, directives, etc can be used by others or not</p>
</li>
<li><p>There is two types of module exists in Angular</p>
</li>
<li><p>Root Module: </p>
</li>
<li><p>Every app can have only one root module.</p>
</li>
<li><p>Imports Browser Module</p>
</li>
<li><p>Defined inside app.module.ts</p>
</li>
<li><p>Feature Module</p>
</li>
<li><p>mg generate module product-list</p>
</li>
</ul>
<h2>Services</h2>
<ul>
<li>Services are objects which get initiated only once during the lifetime of an application.</li>
<li>The main objective of the service to share data</li>
<li>Defined using a @Injectable decorator</li>
<li>A function defined inside a service can be invoked from any component or directive by just importing the service.</li>
<li>ng generate service product-service</li>
</ul>
