2:Tad8,<h2>Create certificate signing request using OpenSSL</h2>
<h3>Download &amp; Install OpenSSL</h3>
<ul>
<li>Download OpenSSL from a trustable source, I downloaded from <a href="https://slproweb.com/products/Win32OpenSSL.html">https://slproweb.com/products/Win32OpenSSL.html</a>. Use <a href="https://tecadmin.net/install-openssl-on-windows/">https://tecadmin.net/install-openssl-on-windows/</a> for detail steps.</li>
<li>Install from Exe or MSI</li>
<li>Set installed binary path in the environment&#39;s  variable.</li>
</ul>
<h3>Prepare config file</h3>
<p>You can pass all this information as command line parameter but using configuration file removes the complexity of dealing with UTF8 characters if there is any, in your subject details.</p>
<blockquote>
<p>[req]  </p>
<p>prompt = no<br>distinguished_name = dn<br>req_extensions = ext<br>[dn]  </p>
<p>CN = domain name # Site description<br>O = My company # My company<br>OU = My dept # My dept<br>L = Budigere, Bangalore # India<br>C = IN<br>[ext]  </p>
<p>subjectAltName = DNS:example.com,DNS:*.example.com</p>
</blockquote>
<p>Command to Generate Certificate Signing Request  </p>
<blockquote>
<p>openssl req -new -utf8 -nameopt multiline,utf8 -config your-url.cnf -newkey rsa:2048 -nodes -keyout your-url.key -out your-url.csr</p>
</blockquote>
<h3>Review your CSR</h3>
<ul>
<li>Open CSR file in text editor (notepad, notepad++ etc.)</li>
<li>Copy whole text from the file</li>
<li>Open <a href="https://www.entrust.net/ssl-technical/csr-viewer.cfm">https://www.entrust.net/ssl-technical/csr-viewer.cfm</a> and paste your content into the CSR Viewer box.</li>
</ul>
<h3>Best Practices</h3>
<ul>
<li><p>Key size must be 2048 minimum</p>
</li>
<li><p>Always use -utf8 option with openssl to avoid issues with unicode characters (multi lingual support)</p>
</li>
<li><p>Signature algorithm must be sha256 or greater</p>
</li>
<li><p>Common name must be in lower case</p>
</li>
<li><p>Do not repeat common name value in the san entry field</p>
</li>
<li><p>Each time you run the command it creates a new keypair. After the certificate is signed both the certificate and the keypair will need to be updated on the server. So keep the generated key safely with you.</p>
</li>
<li><p>If you are generating a certificate for multiple domains configure them like below in config (.cnf) file. It improves readability. When the CSR is generated it gets concatenated.  </p>
<blockquote>
<p>subjectAltName = @alt_names [alt_names]<br>DNS.1 = domain1.app<br>DNS.2 = domain2.app<br>DNS.3 = domain3.app<br>DNS.4 = domain4.app<br>DNS.5 = domain5.app</p>
</blockquote>
</li>
</ul>
<h3>Command to convert certficate to PFX format</h3>
<blockquote>
<p>openssl pkcs12 -export -out x.pfx -inkey x.key -in x.crt</p>
</blockquote>
