2:T716,<h1>What is a compilation in Angular?</h1>
<p>An Angular application consists mainly of components and their HTML templates. Because the components and templates provided by Angular cannot be understood by the browser directly, Angular applications require a compilation process before they can run in a browser. Angular provides two types of compilation:</p>
<h2>JIT</h2>
<ul>
<li>Compiles your app in the browser at runtime.</li>
<li>Development --&gt; Production --&gt; Browser download App --&gt; Angular Parses &amp; Compiles Templates (To JavaScript). The application compiles inside the browser during runtime.</li>
<li>ng build, ng serve</li>
</ul>
<h2>AOT</h2>
<ul>
<li>It reduces the download size of the Angular framework.</li>
<li>Compiles your app and libraries at build time. As code is already compiled browser load executable code (HTML + JS + CSS) and renders the application immediately, which leads to fewer ajax request &amp; reduce application load time because if faster rendering.</li>
<li>Errors can be detected at build time in advance, which helps in minimizing errors.</li>
<li>Development --&gt; Angular Parses &amp; Compiles Templates (To JavaScript) --&gt; Production --&gt; Browser download compiled App (HTML + Javascript). The application compiles during build time. Leads to faster rendering.</li>
<li>With compiled code on the client-side no need to read templates and no risky client-side HTML or Javascript evaluation, so fewer opportunities for injection attacks.</li>
<li>ng build --aot, ng serve --aot</li>
<li>AOT is the default compilation scheme from Angular 9 onwards</li>
<li>The type of compilation (JIT or AOT) depends on the value of the AOT property in your build configuration specified in angular.json. By default, AOT is set to true for new CLI apps.</li>
</ul>
