<h2></h2>
<h2>Development Time Best Practices</h2>
<h3>TSLint</h3>
<ul>
<li>run ng lint before raising PR</li>
<li>fix lint issues and test application if there is any breaking changes using the below command</li>
<li>ng lint --fix</li>
</ul>
<h3>Generates classes using Angular CLI</h3>
<ul>
<li>ng new <project name></li>
<li>ng g c components/products</li>
</ul>
<h3>Avoid Duplicate</h3>
<ul>
<li>HTML --&gt; Use components, Directive, Template</li>
<li>TS --&gt; functions, providers or components</li>
<li>CSS--&gt; use global &amp; component-specific CSS, use SCSS</li>
</ul>
<h3>Use Lazy loading</h3>
<ul>
<li>Instead of loading all components of the application, load only the module which are required by specific use case at run time.</li>
<li>Work well only if application designed well to divide in multiple modules</li>
<li>Help in reducing initial load time</li>
</ul>
<h3>Recommended Plug-ins</h3>
<ul>
<li>Auto Import</li>
<li>MOVE TS</li>
<li>Material icon theme</li>
<li>Angular 2 TypeScript Snippet</li>
</ul>
<h3>TypeScript Best practices</h3>
<ul>
<li>Write small functions</li>
<li>Turn Strict Checks On</li>
<li>Start pinning your versions in your package.json</li>
<li>Use objects as payloads for functions</li>
</ul>
<h3>Third-Party Packages</h3>
<ul>
<li>Review the third party packages used and try to find better and smaller alternative</li>
<li>Remove unnecessary packages used for small functionality. Try to use native Js or HTML5 features if possible</li>
</ul>
<h3>Use defer &amp; async attribute on external Javascript files</h3>
<ul>
<li>lazyload javascript</li>
</ul>
<h3>Avoid complex computations in the template</h3>
<ul>
<li>Leverage the use of pure pipes thereby taking advantage of Angular caching and hence avoiding duplicate operations </li>
<li>If the use of pipe is not possible, see the opportunity to pre-calculate the values and then directly bind values instead of calling the component method in the template.</li>
</ul>
<h3>Progressive Web Apps</h3>
<ul>
<li>Offer offline capabilities to the application</li>
<li>Near-native app experience</li>
</ul>
<h3>Avoid using console.log</h3>
<ul>
<li>It slows down the performance of the application</li>
<li>Create memory leaks issues</li>
<li>In development mode of the browser, it slows down application performance significtly</li>
<li>If you use Webpack you could use the Terser plugin to completely exclude the console.log function calls.</li>
<li>You could define a DEBUG_MODE variable and log conditionally.</li>
<li>Override console.log = function() {};, but it should be your last resort as it may lead to application crash issues, no guaranty if any other library overriding it. In the best case, you can write your own logging library and log conditionally.</li>
</ul>
<h3>Unsubscribing Observables</h3>
<ul>
<li>Observables could create a memory leak issue. So it is better to unsubscribe them when they are not needed anymore. </li>
<li>Unsubscribing explicitly is required when a subscription is created inside a component which is destroyed before the observable completes.</li>
</ul>
<p> </p>
<h3>Observable share() operator</h3>
<ul>
<li>If you have subscribed the observable at multiple locations/components, then each subscription will try to produce the data even though the data is duplicate. </li>
<li>We can avoid the processing of the duplicate data across subscriptions using the “share()” operator.</li>
</ul>
<h3>Global variables</h3>
<ul>
<li><p>The variables defined in the global scope won’t be cleared until the window is reloaded or tab is closed thus resulting in the memory leak </p>
</li>
<li><p>Instead of global variables</p>
</li>
<li><p>Use shared service </p>
</li>
<li><p>Put the global variables in a file an export them, use an import statement &quot;import * as myGlobals from &#39;globals&#39;&quot; in other files</p>
</li>
</ul>
<h3>Short digest Cycle</h3>
<ul>
<li>Your application’s performance will become slow if your app contains a longer digest cycle</li>
<li>Using fewer watchers reduces the digest cycle &amp; improves the speed of your application</li>
</ul>
<h3>Reduce usage of ng-repeat</h3>
<ul>
<li>use alternatives like $interpolate provider, if possible.</li>
</ul>
<h3>Performance &amp; Testing tools</h3>
<ul>
<li>Perform protractor and Batarang scan at the end of every sprint and add assessment results as a technical story in the next sprint</li>
</ul>
<h2>Build Time Best Practices</h2>
<h3>Use <a href="https://github.com/typicode/husky">Husky</a> or <a href="https://git-scm.com/book/en/v2/Customizing-Git-Git-Hooks">Git Hooks</a> or <a href="https://pre-commit.com/">pre-commit</a></h3>
<h3></h3>
<ul>
<li>NPM package that lets you run a set of commands or script before any git action</li>
<li><a href="https://dev.to/ajmaurya/set-up-eslint-prettier-and-pre-commit-hooks-using-husky-for-wordpress-from-scratch-1djk">Configure strict TSLint, prettier hooks with git add action</a></li>
</ul>
<h3><a href="/2020/11/aot-vs-jit-compilation-in-angular.html">Use AOT compilation</a></h3>
<h3>Use Tree-shaking</h3>
<ul>
<li>To remove unused code</li>
</ul>
<h3>Uglify</h3>
<ul>
<li>Uglify does the code transformations like mangling, removal of whitespace, removal of comments etc.</li>
<li>for webpack use uglify plugin with angular-cli, specify the prod&#39; flag to perform the qualification process.</li>
</ul>
<h3>WebPack</h3>
<ul>
<li>Help in reducing build size</li>
</ul>
<h3>Prod flag</h3>
<ul>
<li>specify the &#39;prod&#39; flag in the angular-cli</li>
<li>It enables various build optimization like uglify, AOT, removal of sourcemaps, service workers, and producing a smaller build size.</li>
</ul>
<h3>Build optimizer flag</h3>
<ul>
<li>suggested for a production build</li>
<li>It disables the vendor chunk and helps in reducing build size.</li>
</ul>
<h3>Server Side rendering</h3>
<ul>
<li>Pre-render application on the server and serve static content to the client</li>
<li>Helps in reducing the load time of the application</li>
<li>Use Angular Universal to perform server-side rendering</li>
</ul>
<h3>GZip compression</h3>
<ul>
<li>Reduce the size of the response body, result in low latency</li>
</ul>
<h3>Compressing images</h3>
<ul>
<li>Usually, UX team provides images in high resolution which is not good to use in web applications as it makes the application too heavy. </li>
<li>Ensure to compress images without losing quality</li>
</ul>
<h3>enableProdMode</h3>
<ul>
<li> Invoking “enableProdMode()” avoids Angular from performing additional checks for change detection.</li>
<li>import {enableProdMode} from &#39;@angular/core&#39;;
if (ENV === &#39;production&#39;) {
enableProdMode();
}</li>
</ul>
